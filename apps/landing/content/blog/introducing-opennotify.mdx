## Проблема, которую мы решаем

Каждый разработчик в Узбекистане, работающий с уведомлениями, сталкивается с одной и той же проблемой: **слишком много разрозненных API**.

Хотите отправить SMS? Интегрируйтесь с Eskiz. Нужен Telegram? Отдельный бот и API. Email? SendGrid или собственный SMTP. Push-уведомления? Firebase. WhatsApp? Ещё один API.

В результате код приложения превращается в спагетти из разных SDK, форматов запросов и механизмов обработки ошибок.

## Решение: Один API для всего

OpenNotify объединяет все каналы уведомлений в **единый, простой API**:

```typescript
import { OpenNotify } from "@opennotify/node-sdk"

const client = new OpenNotify({ apiKey: "sk_live_..." })

// Отправка через любой канал — один и тот же код
await client.send({
    channel: "auto", // или "sms", "telegram", "email"
    recipient: "+998901234567",
    message: "Ваш код подтверждения: 123456"
})
```

## Ключевые возможности

### Smart Routing

Самая мощная функция OpenNotify — умная маршрутизация. Система автоматически выбирает оптимальный канал:

- Получатель подключен к вашему Telegram боту? Отправляем туда (бесплатно)
- Нет Telegram? Пробуем Email
- Критически важное сообщение? SMS с гарантированной доставкой

Результат: **экономия до 70%** на уведомлениях без потери доставляемости.

### Локальные провайдеры

Мы глубоко интегрированы с **провайдерами Узбекистана**:

- Eskiz — крупнейший SMS-агрегатор
- PlayMobile — надежная доставка
- GetSMS — конкурентные цены

Вы подключаете свой аккаунт у провайдера — мы предоставляем единый API для управления.

### Аналитика в реальном времени

Отслеживайте каждое уведомление:

- Статусы доставки через webhooks
- Детальная аналитика по каналам
- Учет расходов и экономии
